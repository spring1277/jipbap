# 🍚 우리집 집밥 레시피

가족이 좋아하는 메뉴 · 내가 정리하는 레시피 · 재료 기반 추천과 검색을 담은 **설치형 웹앱(PWA)** 입니다.
서버가 필요 없어 GitHub Pages로 무료 배포할 수 있고, 폰에 설치하면 앱처럼 열리며 오프라인에서도 동작합니다.

## 기능
- **가족·식단** — 가족 구성원별로 좋아하는 메뉴 모아보기, 이번 주 요일별 식단표(아침/점심/저녁) 짜기
- **레시피** — 사진·재료·순서·태그·메모를 담은 나만의 레시피 정리 (분류별 필터)
- **검색·추천**
  - 내 레시피에서 이름·재료·태그로 검색
  - 냉장고 재료를 입력하면 만들 수 있는 메뉴 추천(준비율 %, 부족한 재료 표시)
  - 만개의레시피·우리의식탁·유튜브·구글로 웹 검색 이동
- **더보기** — 백업 내보내기/불러오기(JSON), 설치·배포 안내

데이터는 기기 안(브라우저 IndexedDB)에 저장됩니다. 계정이 필요 없습니다.

## 폰에 설치하기
1. 배포한 주소를 폰 브라우저로 엽니다.
2. **아이폰(Safari)**: 공유 버튼 → "홈 화면에 추가"
   **안드로이드(Chrome)**: 메뉴 ⋮ → "앱 설치" 또는 "홈 화면에 추가"

## GitHub Pages로 배포하기
1. GitHub에서 새 저장소를 만듭니다 (예: `jipbap`).
2. 이 `jipbap-app` 폴더 안의 파일들을 저장소 **루트**에 올립니다.
   (또는 폴더째 올린 뒤 아래 Pages 설정에서 폴더를 지정)
3. 저장소 **Settings → Pages** 로 이동합니다.
4. **Source**: `Deploy from a branch`, **Branch**: `main` / 루트(`/`) 선택 후 Save.
5. 잠시 뒤 생성되는 주소(`https://<사용자명>.github.io/<저장소명>/`)를 폰에서 엽니다.

> 파일을 수정해 다시 배포할 때는 `sw.js` 안의 `CACHE_VERSION` 숫자(`jipbap-v1` → `v2` …)를 올리면
> 이전 캐시가 자동으로 정리되어 항상 최신 버전이 열립니다.

## 로컬에서 미리보기
정적 파일이므로 아무 정적 서버로나 열 수 있습니다.
```bash
cd jipbap-app
python -m http.server 8080
# 브라우저에서 http://localhost:8080
```
(파일을 `file://`로 바로 열면 서비스워커·seed 로딩이 제한되니 위처럼 서버로 여세요.)

## 폴더 구조
```
jipbap-app/
├─ index.html            # 화면 뼈대
├─ styles.css            # 스타일(모바일 우선)
├─ app.js                # 앱 로직 (IndexedDB, 렌더, 검색/추천)
├─ manifest.webmanifest  # PWA 설치 정보
├─ sw.js                 # 서비스워커(오프라인 캐시)
├─ data/seed-recipes.json# 첫 실행 시 들어가는 예시 레시피
└─ icons/                # 앱 아이콘
```
